Prevent Autofill

There are scenarios where autofill is inappropriate and should be disabled, like when the expected data for a form is unrelated to the browser's user. These situations tend to occur in business applications (like a data entry form), but also exist in consumer-facing products (like a two-factor auth code, or an incident report form).

There is an unfortunate arms-race between browser developers and website developers over the ability to disable autofill. There are many email chains and bug reports that discuss the motivations at length, so it suffices to say that when a developer considers modifying autofill behavior, they must consider both usability for the end user and the application's intended workflow.

HTML Standard

The standard way to disable autofill is to set the autocomplete attribute to "off". This is almost universally disrespected by browsers, especially in login fields.
